## 二、核心问题与解决方案

### 问题 1：SQLite Migration 缺失 → 迁移测试持续失败

**现象**  
- `TestSqliteMigrations` 测试每次运行都失败  
- 错误信息：`There are pending model changes for Sqlite`  

**根因分析**  
EF Core 模型变更（新增 `ApkgUpload` 实体）仅为 MySQL 生成了迁移脚本：
- ✅ MySQL: `src/Aiursoft.Apkg.MySql/Migrations/20250526080057_AddApkgUpload.*`
- ❌ SQLite: **缺失对应迁移** → CI 测试检测到数据库 schema 不同步

**解决方案**  
```bash
# 为 SQLite 补齐迁移脚本
cd /path/to/apkg
dotnet ef migrations add AddApkgUpload \
  --project src/Aiursoft.Apkg.Sqlite \
  --startup-project src/Aiursoft.Apkg
```

**输出物**  
- `src/Aiursoft.Apkg.Sqlite/Migrations/20250526080057_AddApkgUpload.cs`  
- `src/Aiursoft.Apkg.Sqlite/Migrations/ApkgDbContextModelSnapshot.cs` (更新)

**关键点**  
- 多数据库项目必须为每个 DbContext 生成迁移（MySQL、SQLite、InMemory）  
- CI 的 `MigrationTests.cs` 是一个重要的保障机制，会自动检测此类漂移

**重点修复：StreamContent 初始化规则** 

❌ **错误做法**（ReSharper 警告）：
```csharp
using var content = new StreamContent(stream)
{
    Headers = { ContentType = new MediaTypeHeaderValue("application/octet-stream") }
};
```
问题：对象初始化在 using 语句外执行，若初始化时发生异常，资源无法正确释放。

✅ **正确做法**（提取 factory）：
```csharp
private static StreamContent CreateApkgFileContent(Stream stream)
{
    var content = new StreamContent(stream);
    content.Headers.ContentType = new MediaTypeHeaderValue("application/octet-stream");
    return content;
}

// 使用
using var content = CreateApkgFileContent(stream);
```

Razor section 警告（Upload.cshtml）：

### 3.2 ReSharper 规则速查表

| 规则 | 触发条件 | 修复方案 |
|------|--------|--------|
| `UnusedVariable` | 赋值后未使用 | 移除赋值或使用变量 |
| `RedundantTypeCastExpression` | `(Type)value` 当 value 已是 Type | 删除转换 |
| `InitializerInlineWithUsing` | `new Type() { Props = value }` 在 using 外 | 提取 factory 方法或使用内联初始化 |
| `Razor.SectionNotResolved` | Razor 找不到 layout 中的 section | 加 `@* ReSharper disable once *@` 抑制 |

**全局配置**：`.editorconfig` 或 ReSharper 设置面板（IDE → Code Inspection）

## 六、常见陷阱与最佳实践

### 陷阱 1：忘记同步 SQLite 迁移

**后果**：CI 测试失败，阻塞 merge  
**预防**：每次修改 Entities 后立即运行 `dotnet test`，检查 MigrationTests

### 陷阱 2：Razor section 警告误判

**现象**：Upload.cshtml 报 `section not resolved`，但实际存在于 Layout  
**原因**：ReSharper Razor 语言服务偶尔出现假阳性  
**处理**：用 `@* ReSharper disable once *@` 抑制（参考 LocalPackages/Upload.cshtml）

### 陷阱 3：测试间 cookie 污染

**现象**：第二个测试看到第一个测试的登录状态  
**原因**：跨测试共享 HttpClient  
**解决**：在 SetupTestContext 中为每个测试创建新 client  

### 陷阱 4：Redirect 路径混淆

**现象**：重定向期望 `/ApkgUploads/Index`，实际是 `/ApkgUploads`  
**原因**：ASP.NET Core 约定路由，Index 是默认 action  
**解决**：检查 controller route + action 属性，调整测试预期
